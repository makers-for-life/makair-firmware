59 uint16_t PressureValve::openLinear(uint16_t p_command) {
60  positionLinear = p_command;
61  // The p_command is in [ 0 ; 125 ], but the valve is only effective in [30; 100]
62  // So lets make it between 30 and 100
63  // A x10 multiplier is used here for better precision with integer calculations
64  // The value of intermediateValue will consequently be in [300 ; 1000]
65  uint32_t intermediateValue = (((static_cast<uint32_t>(p_command) * 75u) / 125u) + 30u) * 10u;
66 
67  /* An order 3 polynom is used to correct the non linearity of the valve.
68  To find this polynom, the following experimental protocol was used:
69  - Run the blower max speed during the full experience
70  - Plug inspiratory output on expiratory input
71  - For each openning value of the valve, wait 5s for stabilisation and measure Flow
72 
73  This gives a csv with:
74  openningValue[0;125], flow (mL/min)
75  0, 48137
76  10, 47780
77  ...
78  125, 127
79 
80  Then we calculated for each openning value the "linear target flow" with a basic linear
81  equation:
82  openningValue[0;125], flow (mL/min), targetFlow (mL/min)
83  0, 48137,48137
84  10, 47780, 44296
85  ...
86  125, 127, 0
87 
88  Then we manually found for each openning value, the openning value that should have been used
89  to reach the targetFlow:
90  openningValue[0;125], flow (mL/min), targetFlow (mL/min),
91  correctedOpenningValue[0;125] 0, 48137,48137, 30 10, 47780, 44296, 55.2
92  ...
93  125, 127, 0, 100
94 
95  Then when made an order 3 regression between correctedOpenningValue and openningValue. This
96  is the relation below:
97  */
98  command = (uint16_t)(
99  (((76u * intermediateValue) / 10u)
100  - (((985u * intermediateValue) * intermediateValue) / 100000u)
101  + (((((44u * intermediateValue) * intermediateValue) / 1000u) * intermediateValue)
102  / 10000u)
103  - 1140u)
104  / 10u);
105 
106  return command;
107 }
